About Us

  • ECloud-Based Vehicle Data Monitoring System (CBVDMS) aims to collectively identify problems and malfunctions in a car’s system, get real time overall performance feedback and provide significant description from the Diagnostic Trouble Codes (DTS) gathered. The user (car owner) should have the minimum requirements for the project to work—a car with an On-Board Diagnostics (OBD) II port, having an OBD Bluetooth adapter and Raspberry Pi on the hardware side, and a desktop or phone to access the website. The target audience and users will be car owners, car mechanics, car repair shops, car proprietors and even transportation businesses. With the CBVDMS, users can keep track of their vehicle data anywhere and anytime.

Services

  • Real-time Data Acquisition

    The system retrieves the OBD Parameter IDs (PIDs) real time, allowing the user to observe changes on vehicle data (Engine RPM, Throttle Position, etc.).

  • Current Status Report

    Enables users to ping vehicle status.

  • Error History Log

    Logs Data Trouble Codes (DTCs) that has been encountered by the car.
